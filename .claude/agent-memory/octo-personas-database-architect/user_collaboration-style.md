The user runs a disciplined "align before building" workflow (global CLAUDE.md): grill/brainstorm to a shared understanding before any code, simplicity-first, surgical changes, one confirmed step at a time.

For deliverables they explicitly ask for: SHORT, skimmable, tables/lists over prose, with concrete worked examples. They flag "do NOT over-engineer" and want simplifications called out.

**How to apply:** lead with tables/lists; include at least one worked numeric example when explaining a mechanism; explicitly mark what's deferred/kept-simple; don't expand scope beyond the ask. When acting as a specialist persona, stay in that lane (e.g. data-layer storage vs. runtime algorithm) and note where another persona owns adjacent work.
